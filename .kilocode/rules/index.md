# Agent Operational Rules Index

**Version**: 1.0
**Last Updated**: 2026-02-18

This directory contains modularized operational rules for AI agents in the kilocode system. Each file focuses on a specific aspect of agent behavior and collaboration protocols.

## Core Rules Files

- **Purpose**: Defines the fundamental identity and collaboration principles for AI agents
- **Content**: Intellectual honesty, operating principles, approval protocols, final reminders
- **Key Sections**:
  - Core Identity & Collaboration Philosophy
  - Approval & Intellectual Honesty Protocol
  - Final Reminders

### [directory-structure.md](directory-structure.md)
- **Purpose**: Establishes standards for project organization and documentation artifacts
- **Content**: Directory structure standards, critical documentation artifacts by phase
- **Key Sections**:
  - Directory Structure Standard
  - Critical Documentation Artifacts (Planning, Design, Implementation, Testing phases)

### [knowledge-base.md](knowledge-base.md)
- **Purpose**: Defines protocols for knowledge base management and decision-making
- **Content**: KB query/update patterns, documentation standards, when to use KB
- **Key Sections**:
  - Knowledge Base Protocol
  - Query/Update Timing
  - Documentation Standards

### [quality-framework.md](quality-framework.md)
- **Purpose**: Establishes quality assurance criteria and role-specific evaluation metrics
- **Content**: Self-assessment requirements, role-specific quality criteria, iteration protocols
- **Key Sections**:
  - Quality Assurance Framework
  - Role-Specific Criteria (Architect, Developer, QA, DevOps)
  - Role Adaptation Matrix

### [session-protocol.md](session-protocol.md)
- **Purpose**: Defines protocols for session continuity and progress tracking
- **Content**: Session start/end checklists, progress tracking standards, blocker management
- **Key Sections**:
  - Session Continuity Protocol
  - Progress Tracking Standard
  - Session Management Procedures

## Usage Guidelines

1. **Read Order**: Start with `core-philosophy.md` for foundational principles, then `session-protocol.md` for operational procedures
2. **Reference**: Use this index to quickly locate relevant rules for specific scenarios
3. **Updates**: When rules are updated, ensure cross-references remain valid
4. **Integration**: These rules work together - decisions in one area may reference protocols in another
